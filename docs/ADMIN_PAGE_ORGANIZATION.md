## 📊 页面组织方式

### 当前页面结构

```
pages/
├── 1_🏠_首页.py                    (所有用户)
├── 2_📤_视频上传.py                 (所有用户)
├── 3_📈_行为分析.py                 (所有用户)
├── 4_💡_教学建议.py                 (所有用户)
├── Admin_0_👑_管理员中心.py         (仅管理员) ⭐ 新增
├── 5_👤_用户管理.py                 (仅管理员)
└── 6_📅_课表管理.py                 (仅管理员)
```

### 命名说明

- **通用页面**：使用数字前缀（1-4），按功能顺序排列
- **管理员页面**：使用 `Admin_` 前缀，在侧边栏中会集中显示
  - `Admin_0_`：管理员中心（仪表盘）
  - `5_`：用户管理（保持原编号，避免重命名问题）
  - `6_`：课表管理（保持原编号，避免重命名问题）

**注意**：由于Streamlit不支持子目录，使用命名约定实现分组效果。

## 🎯 使用方式

### 对于管理员用户

1. **登录后**：侧边栏会显示"👑 管理员"分组
2. **访问管理员中心**：
   - 方式1：点击侧边栏的"👑 管理员中心"按钮
   - 方式2：在Streamlit页面列表中选择 `Admin_0_👑_管理员中心`
3. **快速操作**：在管理员中心页面可以一键跳转到用户管理或课表管理

### 对于普通教师用户

- 侧边栏只显示通用页面（首页、视频上传、行为分析、教学建议）
- 不会显示管理员页面入口
- 如果直接访问管理员页面URL，会显示权限错误

## 🔐 权限控制

### 实现方式

1. **页面级权限检查**：
   ```python
   if current_user.get('role') != 'admin':
       st.error("⚠️ 只有管理员可以访问此页面")
       st.stop()
   ```

2. **侧边栏菜单**：
   - 根据 `role` 动态显示管理员菜单
   - 只有 `role == 'admin'` 时才显示管理员分组

3. **API权限**：
   - 后端API已有权限检查（JWT Token + 角色验证）
   - 前端只是UI层面的控制
