# 行为分析模块改进方案（无个体识别版本）

## 📋 改进目标

在不支持学生个体识别、没有座位信息的情况下，最大化利用现有数据（检测框位置、行为类别、时间戳）进行深度分析。

---

## 🎯 改进方案

### 一、空间区域分析（利用检测框位置）

#### 1.1 画面区域划分
**思路：** 将视频画面划分为多个区域，分析不同区域的行为分布

**实现方式：**
- 将画面分为：**前中后排**（垂直3等分）或 **左右侧**（水平2等分）
- 根据检测框的 `bbox` 坐标判断所属区域
- 统计每个区域的行为密度

**需要的数据：**
- ✅ 已有：检测框坐标 `bbox: [x1, y1, x2, y2]`
- ✅ 已有：视频分辨率（用于计算区域边界）

**输出：**
- 区域行为热力图
- 前后排参与度对比
- 左右侧注意力分布

**示例：**
```
前排区域：举手次数较多，参与度高
后排区域：低头行为较多，需要关注
```

---

### 二、行为密度热力图（空间分布可视化）

#### 2.1 行为空间分布
**思路：** 将检测框位置映射到画面坐标，生成行为密度热力图

**实现方式：**
- 统计每个像素区域的行为检测次数
- 使用高斯核平滑生成热力图
- 不同行为类别用不同颜色

**需要的数据：**
- ✅ 已有：检测框坐标
- ✅ 已有：行为类别

**输出：**
- 举手行为热力图（显示举手集中区域）
- 低头行为热力图（显示注意力分散区域）
- 讨论行为热力图（显示互动活跃区域）

**价值：**
- 直观显示课堂"活跃区"和"冷区"
- 帮助教师调整教学策略

---

### 三、真实相关性分析（替代预定义规则）

#### 3.1 皮尔逊相关系数计算
**思路：** 使用真实统计方法计算教师行为与学生行为的相关性

**实现方式：**
```python
# 使用 numpy/scipy 计算
from scipy.stats import pearsonr

# 提取时间序列
teacher_guide_series = [窗口1的引导次数, 窗口2的引导次数, ...]
student_discuss_series = [窗口1的讨论次数, 窗口2的讨论次数, ...]

# 计算相关系数
correlation, p_value = pearsonr(teacher_guide_series, student_discuss_series)
```

**需要的数据：**
- ✅ 已有：时间线数据（每个窗口的行为计数）

**输出：**
- 真实的相关性系数（-1到1）
- 显著性检验（p值）
- 滞后时间分析（教师行为后多久学生响应）

**改进点：**
- 从预定义规则 → 真实数据驱动
- 更准确反映实际教学效果

---

### 四、行为模式识别

#### 4.1 典型教学模式识别
**思路：** 基于行为时间序列，自动识别典型的教学模式

**实现方式：**
- 使用聚类算法（K-means）对时间窗口进行分类
- 识别出：讲授模式、互动模式、练习模式、总结模式
- 分析每种模式的时长占比和效果

**需要的数据：**
- ✅ 已有：时间线数据（每个窗口的行为向量）

**输出：**
- 教学模式时间线（自动标注）
- 模式占比统计
- 模式转换频率

**价值：**
- 自动识别教学节奏
- 评估教学模式多样性

---

### 五、动态异常阈值

#### 5.1 自适应阈值
**思路：** 基于历史数据或同类型课程，动态调整异常检测阈值

**实现方式：**
```python
# 计算历史平均值和标准差
mean_bowhead_rate = 计算历史平均低头率()
std_bowhead_rate = 计算历史标准差()

# 动态阈值 = 平均值 + 2倍标准差
threshold = mean_bowhead_rate + 2 * std_bowhead_rate
```

**需要的数据：**
- ⚠️ 需要：历史课程数据（同一班级或同类型课程）
- ✅ 已有：当前课程数据

**输出：**
- 基于数据的异常阈值
- 更准确的异常检测

---

### 六、行为转换分析

#### 6.1 行为状态机
**思路：** 分析行为之间的转换频率和模式

**实现方式：**
- 统计行为转换：举手 → 回答、讨论 → 书写等
- 构建行为转换矩阵
- 识别典型转换路径

**需要的数据：**
- ✅ 已有：时间线数据（可以推断行为转换）

**输出：**
- 行为转换矩阵（桑基图）
- 典型转换路径
- 转换频率统计

**价值：**
- 了解课堂行为流转
- 识别教学环节衔接

---

### 七、多节课对比分析

#### 7.1 班级整体趋势
**思路：** 对比同一班级多节课的行为数据，分析趋势

**实现方式：**
- 选择2-3节课进行对比
- 对比指标：互动率、专注度、教学模式分布
- 时间序列对比图

**需要的数据：**
- ✅ 已有：每节课的分析数据
- ⚠️ 需要：班级信息（已有 `class_name` 字段）

**输出：**
- 多节课对比图表
- 趋势分析报告
- 改进效果评估

---

### 八、峰值时段分析

#### 8.1 关键时段识别
**思路：** 识别课堂中的"黄金时段"和"低谷时段"

**实现方式：**
- 计算每个时间窗口的综合活跃度
- 识别峰值时段（互动最活跃）
- 识别低谷时段（需要关注）

**需要的数据：**
- ✅ 已有：时间线数据

**输出：**
- 活跃度曲线
- 峰值时段标注
- 低谷时段建议
