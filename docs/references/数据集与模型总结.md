## 🤖 模型类型与架构

### 统一模型架构

**所有4个模型都使用相同的架构：**

```python
模型类型: Faster R-CNN
骨干网络: vHeat (MMDET_VHEAT)
检测头: FPN (Feature Pyramid Network)
```

### 详细配置

#### 1. 模型类型：Faster R-CNN

**为什么选择 Faster R-CNN？**
- ✅ **两阶段检测器**：精度高，适合复杂场景
- ✅ **成熟稳定**：MMDetection 中实现完善
- ✅ **适合多类别**：支持多类别目标检测
- ✅ **与 vHeat 兼容**：可以无缝替换 backbone

#### 2. 骨干网络：vHeat

**vHeat 特点：**
- ✅ **高效计算**：复杂度 O(N^1.5)
- ✅ **全局感受野**：捕获图像全局语义
- ✅ **性能优异**：在 COCO 上超越 Swin Transformer
- ✅ **低内存占用**：适合高分辨率图像

**配置示例：**
```python
backbone=dict(
    type='MMDET_VHEAT',
    arch='tiny',  # 或 'small', 'base'
    drop_path_rate=0.2,
    out_indices=(0, 1, 2, 3),
)
```

#### 3. 特征金字塔网络：FPN

**FPN 作用：**
- 多尺度特征融合
- 提升小目标检测能力
- 增强特征表达能力

## 🔄 训练与推理方式

### 训练配置

#### 图像分辨率
- **训练分辨率：** 704 x 576
- **保持宽高比：** `keep_ratio=True`
- **数据增强：** 
  - 随机翻转
  - 随机缩放
  - 颜色抖动（如配置）

#### 训练参数（典型配置）
```python
# 批次大小
batch_size = 16  # 根据GPU内存调整

# 学习率
lr = 0.02
lr_schedule = '1x'  # 或 '3x'

# 训练轮数
epochs = 12  # 1x schedule
# epochs = 36  # 3x schedule

# 优化器
optimizer = dict(type='SGD', lr=0.02, momentum=0.9, weight_decay=0.0001)
```

#### 训练数据格式
- **COCO 格式**（comprehensive_scene 数据集）
- **YOLO 格式**（其他3个数据集）

### 推理配置

#### 图像预处理管道

**所有模型使用相同的推理管道：**

```python
test_pipeline = [
    dict(type='LoadImageFromFile', backend_args=backend_args),
    dict(type='Resize', scale=(704, 576), keep_ratio=True),  # ⚠️ 必须与训练一致！
    dict(type='LoadAnnotations', with_bbox=True, with_mask=False),
    dict(
        type='PackDetInputs',
        meta_keys=('img_id', 'img_path', 'ori_shape', 'img_shape', 'scale_factor')
    )
]
```

**关键点：**
- ✅ **分辨率必须一致**：推理时使用 `(704, 576)`，与训练时相同
- ✅ **保持宽高比**：`keep_ratio=True` 避免图像变形
- ✅ **返回缩放因子**：`scale_factor` 用于将检测框映射回原图

#### 推理流程

```
输入图像 (任意分辨率)
    ↓
Resize (704, 576, keep_ratio=True)
    ↓
模型推理 (Faster R-CNN + vHeat)
    ↓
检测结果 (bbox, score, label)
    ↓
坐标映射回原图 (使用 scale_factor)
    ↓
输出检测框 (原图坐标系)
```

### 多模型协同推理

#### 推理策略

**并行推理：**
- 4个模型同时对同一帧进行推理
- 每个模型输出各自的检测结果

**结果融合：**
- 跨模型 NMS 去重
- 置信度过滤（默认阈值：0.3）
- 冲突解决（同一目标被多个模型检测）
- 上下文验证（如：`blackboard-writing` 需要 `teacher` 在附近）

#### 推理代码示例

## ⚠️ 重要注意事项

### 1. 分辨率一致性

**训练和推理必须使用相同的分辨率！**

```python
# ✅ 正确
test_pipeline = [
    dict(type='Resize', scale=(704, 576), keep_ratio=True),
    ...
]

# ❌ 错误（会导致检测框错位）
test_pipeline = [
    dict(type='Resize', scale=(1536, 960), keep_ratio=True),
    ...
]
```

### 2. 模型注册

**所有使用 vHeat backbone 的配置文件必须添加：**

```python
custom_imports = dict(
    imports=['detection.model'],
    allow_failed_imports=False
)
```

## 📈 训练建议

### 数据增强策略

#### 对于数据量充足的模型（comprehensive, learning）
- 标准数据增强即可
- 随机翻转、随机缩放

#### 对于数据量较少的模型（discussion, posture）
- **强烈建议使用数据增强：**
  - 随机翻转（水平、垂直）
  - 随机旋转（±15度）
  - 颜色抖动（亮度、对比度、饱和度）
  - 随机裁剪
  - MixUp / CutMix（可选）

### 训练策略

#### 1. 预训练权重
- 使用 ImageNet 预训练的 vHeat 权重
- 或使用 COCO 预训练的检测模型权重

#### 2. 学习率调度
- **1x schedule**: 12 epochs，适合快速实验
- **3x schedule**: 36 epochs，适合追求最佳性能

#### 3. 多尺度训练（可选）
- 训练时使用多尺度输入
- 提升模型对不同分辨率的适应性
