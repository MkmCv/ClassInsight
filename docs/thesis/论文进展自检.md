# 学生对毕业论文（设计）进展情况、存在的问题与主要困难、下一步计划与安排以及执行情况自检自查

## 一、开题以来对毕业设计的完成以及进展情况

### 1. 选题与背景理解
- 明确选题为**课堂行为智能分析系统（ClassInsight）**，面向微格教学场景，为师范生/教师提供视频上传、行为检测、数据分析与教学建议。
- 了解选题背景与意义：利用 AI 与计算机视觉辅助教学反思与技能评估，对师范生实训和教师专业发展具有实际应用价值。
- 对项目所需技术形成初步认识：前后端分离、深度学习目标检测、RESTful API、数据库与权限体系等。

### 2. 开题与文献调研
- 完成开题报告，并与导师讨论、修改与优化。
- 通过学校网站等渠道检索并阅读与**课堂行为分析、微格教学、目标检测（YOLO）、教学评价**等相关的文献与资料。
- 进一步学习并梳理系统涉及的技术路线：Python 全栈开发、FastAPI/Streamlit、YOLO-vHeat 多模型检测、大模型与 Agent 在教学建议中的应用等。

### 3. 技术学习与系统搭建
- 学习并采用 **Python 全栈**技术路线：后端使用 **FastAPI** 构建 RESTful API，前端使用 **Streamlit** 构建多页面应用，数据库使用 **SQLAlchemy（异步）+ SQLite**。
- 完成系统整体架构搭建：前后端分离、JWT 认证、分层设计（API → Service → Model），以及普通端与超级管理员端双应用结构。
- 完成核心业务模块开发与联调：
  - **视频上传与管理**：视频上传、存储与元数据管理。
  - **行为分析**：基于 YOLO-vHeat 多模型对课堂视频进行行为检测与统计分析（讲授、板书、屏幕使用、手势指导等）。
  - **教学建议**：集成通义千问与 LangGraph，基于分析结果生成个性化教学改进建议。
  - **用户与权限**：用户管理、角色区分（教师/管理员/超级管理员）、课表管理。
  - **安全与运维**：登录安全配置、登录失败摘要、模型管理等超级管理员功能。
- 编写并维护技术文档（如《技术框架与范式总结》《微格课堂分析方案》等），便于后续论文撰写与答辩说明。

---

## 二、在此阶段中存在的问题与主要困难

### 1. 工程经验与架构熟悉度
- 此前较少独立完成类似规模的**前后端分离 + AI 能力集成**的完整系统，对整体架构设计、模块划分与接口约定仍在实践中不断优化。
- 对 FastAPI 异步编程、Streamlit 状态管理与多页面组织等具体用法，从零到可用的过程中存在一定学习与试错成本。

### 2. 算法与工程结合
- YOLO-vHeat 多模型检测、结果后处理与业务指标（如各行为时长占比、时序统计）的对接需要同时理解模型输出与业务需求，在指标定义与可视化呈现上经过多轮调整。
- 大模型与 LangGraph 在工作流设计、提示词与结果解析方面的应用尚不熟练，教学建议模块的稳定性和可解释性仍有改进空间。

### 3. 时间与优先级
- 开发、调试与文档编写并行时，有时难以兼顾论文写作进度，需要更有计划地分配时间。

## 三、下一步计划与安排

### 1. 系统完善与稳定性
- 继续巩固 FastAPI、Streamlit、SQLAlchemy 等技术的使用，优化接口设计与错误处理，提升系统稳定性和可维护性。
- 根据测试与使用反馈，完善行为分析结果展示与教学建议的表述逻辑，必要时补充单元测试或接口测试。

### 2. 资料补充与论文撰写
- 继续查阅与课堂行为分析、微格教学、目标检测及教学评价相关的文献，在论文中体现文献综述与理论/技术依据。
- 系统学习学校规定的**论文格式与注意事项**，按章节完成初稿（选题背景与意义、需求分析、系统设计、关键技术、实现与测试、总结与展望等），提交指导教师审阅。

### 3. 修改与定稿
- 根据导师意见逐条修改论文内容与格式，经导师审核通过后形成**答辩前最终稿**。
- 整理系统演示流程与关键界面截图，准备答辩陈述与可能的问答。
