# 📖 ClassInsight 用户使用指南

本文档介绍如何使用 ClassInsight 课堂行为分析系统。

---

## 🚀 系统启动

### 1. 启动后端服务

```powershell
# 激活 conda 环境
conda activate vHeat

# 进入后端目录
cd "H:\毕业设计\System\backend"

# 启动服务
python start_server.py
```

启动成功后会显示：
```
工作目录: H:\毕业设计\System\backend
正在启动 ClassInsight 后端服务...
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

### 2. 启动前端服务

**打开新的终端窗口**：

```powershell
# 激活 conda 环境
conda activate vHeat

# 进入前端目录
cd "H:\毕业设计\System\System\frontend"

# 启动 Streamlit
streamlit run app.py
```

启动成功后会自动打开浏览器，或访问 http://localhost:8501

## 👤 用户注册

系统不提供自助注册功能，需要通过 **API 文档页面** 手动注册用户。

### 步骤：

1. **打开 API 文档页面**
   
   浏览器访问：http://localhost:8000/api/docs

2. **找到注册接口**
   
   在页面中找到 `POST /api/v1/auth/register`

3. **点击 "Try it out"**

4. **填写用户信息**
   
   在 Request body 中填入：
   ```json
   {
     "username": "你的用户名",
     "password": "你的密码（至少8位）",
     "email": "你的邮箱@example.com",
     "role": "teacher",
     "unit": "所属学校/单位",
     "class_name": "负责的班级（可选）"
   }
   ```

5. **点击 "Execute" 执行**

6. **查看响应**
   
   成功会返回：
   ```json
   {
     "user_id": 1,
     "username": "你的用户名",
     "message": "注册成功"
   }
   ```

## 🔐 用户登录

1. 访问前端页面：http://localhost:8501
2. 输入注册时的用户名和密码
3. 点击「登录」按钮

登录成功后自动跳转到首页。

## 📚 API 文档说明

### 访问地址

- **Swagger UI**: http://localhost:8000/api/docs
- **ReDoc**: http://localhost:8000/api/redoc
- **健康检查**: http://localhost:8000/health

### API 文档的作用

API 文档（Swagger UI）是一个交互式的接口测试工具，可以：

1. **查看所有接口** - 了解系统提供哪些功能
2. **在线测试接口** - 无需编写代码即可测试 API
3. **查看请求/响应格式** - 了解每个接口需要的参数和返回的数据结构
4. **调试问题** - 当前端出现问题时，可以直接测试后端接口是否正常

### 主要接口模块

| 模块 | 路径前缀 | 功能 |
|------|---------|------|
| 认证 | `/api/v1/auth` | 注册、登录、修改密码 |
| 首页 | `/api/v1/dashboard` | 课程表、指标数据 |
| 视频 | `/api/v1/videos` | 上传、列表、状态查询 |
| 分析 | `/api/v1/analysis` | 行为统计、时间线、异常 |
| 优化 | `/api/v1/optimization` | 建议、雷达图、对比 |

## ⚠️ 常见问题

### Q: 登录时提示"无法连接到服务器"

**A:** 请确认后端服务已启动，检查 http://localhost:8000/health 是否能访问。

### Q: 首页数据显示 N/A

**A:** 说明后端连接异常或没有数据。可以：
1. 检查后端服务是否运行
2. 上传视频并等待处理完成

### Q: 如何修改密码？

**A:** 登录后，在 API 文档页面使用 `PUT /api/v1/auth/password` 接口修改。
