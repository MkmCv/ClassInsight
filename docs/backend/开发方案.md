# 🚀 课堂行为分析系统开发方案

## 📋 开发策略

### ✅ 建议：先设计API接口，再实现后端

**原因：**
1. **理清业务逻辑**：接口设计过程可以帮助梳理数据流和业务需求
2. **前后端并行开发**：接口定义后，前后端可以同时开发，提高效率
3. **便于测试**：可以先使用Postman/curl测试接口，验证后端逻辑
4. **降低返工风险**：避免后端实现后才发现接口设计不合理

---

## 🏗️ 系统架构设计

### 技术栈选择

根据开题报告，建议采用以下技术栈：

```
后端框架：FastAPI（高性能、自动生成API文档）
前端框架：Streamlit（快速原型）或 Vue3/React（完整Web应用）
数据库：SQLite（开发） + PostgreSQL（生产）
认证：JWT + BCrypt
可视化：ECharts（前端） + Matplotlib（后端报告）
任务队列：Celery（异步视频处理）或 简单线程池
```

## 🗄️ 数据库设计

### 核心表结构

#### users 用户表
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,  -- BCrypt哈希
    email VARCHAR(100),
    role VARCHAR(20) NOT NULL,  -- 'teacher' | 'admin'
    unit VARCHAR(100),           -- 单位
    class_name VARCHAR(100),     -- 班级
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### videos 视频表
```sql
CREATE TABLE videos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    filename VARCHAR(255) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    file_size BIGINT,            -- 字节
    duration INTEGER,            -- 秒
    fps REAL,
    resolution VARCHAR(20),      -- '1920x1080'
    class_name VARCHAR(100),
    course_name VARCHAR(100),
    lesson_date DATE,
    status VARCHAR(20) NOT NULL, -- 'uploaded' | 'processing' | 'completed' | 'failed'
    progress REAL DEFAULT 0.0,   -- 0-1
    error_message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### detections 检测结果表（帧级）
```sql
CREATE TABLE detections (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    video_id INTEGER NOT NULL,
    frame_id INTEGER NOT NULL,
    timestamp REAL NOT NULL,     -- 秒
    category_id INTEGER NOT NULL, -- 全局类别ID
    category_name VARCHAR(50) NOT NULL,
    bbox_x1 REAL,
    bbox_y1 REAL,
    bbox_x2 REAL,
    bbox_y2 REAL,
    confidence REAL,
    model_name VARCHAR(50),      -- 'comprehensive' | 'learning' | 'discussion' | 'posture'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (video_id) REFERENCES videos(id),
    INDEX idx_video_frame (video_id, frame_id),
    INDEX idx_video_timestamp (video_id, timestamp)
);
```

#### behavior_summary 行为汇总表（整课聚合）
```sql
CREATE TABLE behavior_summary (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    video_id INTEGER NOT NULL UNIQUE,
    duration INTEGER NOT NULL,
    total_frames INTEGER NOT NULL,
    -- 学生行为统计（JSON格式存储）
    student_behaviors JSON,  -- {"discuss": {"count": 45, "duration": 1200, "percentage": 33.3}, ...}
    -- 教师行为统计
    teacher_behaviors JSON,
    -- 元数据
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (video_id) REFERENCES videos(id)
);
```

#### analysis_reports 分析报告表
```sql
CREATE TABLE analysis_reports (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    video_id INTEGER NOT NULL,
    report_type VARCHAR(50),    -- 'summary' | 'detailed' | 'comparison'
    file_path VARCHAR(500),     -- PDF文件路径
    status VARCHAR(20),          -- 'generating' | 'completed' | 'failed'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (video_id) REFERENCES videos(id)
);
```

## 🔄 开发流程

### 阶段1：接口设计与数据库搭建（第1-2天）

1. **定义API接口规范**
   - 使用OpenAPI/Swagger编写接口文档
   - 创建 `api_spec.yaml` 文件

2. **数据库设计**
   - 创建SQLite数据库
   - 使用SQLAlchemy定义ORM模型
   - 编写数据库迁移脚本

3. **项目结构搭建**
   ```
   System/
   ├── app/
   │   ├── api/
   │   │   ├── v1/
   │   │   │   ├── auth.py      # 认证接口
   │   │   │   ├── videos.py    # 视频处理接口
   │   │   │   ├── analysis.py  # 行为分析接口
   │   │   │   └── optimization.py  # 优化建议接口
   │   ├── core/
   │   │   ├── config.py        # 配置管理
   │   │   ├── security.py      # JWT认证
   │   │   └── database.py      # 数据库连接
   │   ├── models/              # SQLAlchemy模型
   │   ├── services/            # 业务逻辑层
   │   │   ├── video_service.py
   │   │   ├── analysis_service.py
   │   │   └── report_service.py
   │   └── main.py              # FastAPI应用入口
   ├── src/
   │   └── core/
   │       ├── detector.py      # 模型推理（已有）
   │       └── config.py        # 模型配置（已有）
   ├── configs/
   └── requirements.txt
   ```

### 阶段2：核心后端功能实现（第3-5天）

1. **认证模块**
   - 实现JWT token生成与验证
   - 实现BCrypt密码哈希
   - 实现用户注册/登录接口

2. **视频处理模块**
   - 视频上传与存储
   - 异步任务处理（使用Celery或线程池）
   - 视频解码与抽帧
   - 调用MultiModelDetector进行推理
   - 结果存储到数据库

3. **行为分析模块**
   - 时间窗口平滑（10s/1min）
   - 整课聚合统计
   - 异常检测（Z-score/MAD）
   - 相关性分析

### 阶段3：前端界面开发（第6-7天）

1. **使用Streamlit快速原型**（推荐先做）
   - 登录页面
   - 视频上传页面
   - 数据看板（ECharts集成）
   - 报告查看页面

2. **或使用Vue3/React完整前端**（如果需要）
   - 组件化开发
   - 路由管理
   - 状态管理（Vuex/Pinia）

### 阶段4：测试与优化（第8-9天）

1. **单元测试**
   - API接口测试
   - 业务逻辑测试

2. **性能优化**
   - 视频处理异步化
   - 数据库查询优化
   - 缓存策略

3. **部署准备**
   - Docker容器化
   - 环境配置文档
