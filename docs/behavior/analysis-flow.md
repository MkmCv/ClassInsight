# 行为分析模块 - 分析流程详解

## 📋 整体流程概览

```
视频上传 → 视频处理 → AI检测 → 数据聚合 → 统计分析 → 异常检测 → 结果存储
```

---

## 🔍 详细分析流程

### 第一步：视频上传与预处理

1. **用户上传视频**
   - 视频存储在 `storage/videos/` 目录
   - 创建数据库记录，状态为 `pending`

2. **后台任务启动**
   - 异步处理任务 `process_video_task()`
   - 状态更新为 `processing`

3. **视频元数据提取**
   - 使用 OpenCV 读取视频信息：
     - FPS（帧率）
     - 总帧数
     - 分辨率（宽×高）
     - 时长（秒）

---

### 第二步：视频抽帧

**抽帧策略：**
- **抽帧频率**：根据配置 `VIDEO_PROCESS_FPS = 1`（每秒1帧）
- **计算方式**：`frame_interval = fps / VIDEO_PROCESS_FPS`
  - 例如：30fps 的视频，每30帧抽取1帧
  - 例如：25fps 的视频，每25帧抽取1帧

**为什么这样设计？**
- ✅ 平衡分析精度和处理速度
- ✅ 避免重复检测相似帧
- ✅ 1秒1帧足以捕捉行为变化

---

### 第三步：AI 行为检测（核心）

**使用的模型：**
系统使用 **4个 YOLO-vHeat 模型** 进行多模型检测：

#### 1. **学生姿态模型** (`posture`)
- 检测行为：`BowHead`（低头）、`TurnHead`（转头）
- 类别ID：1-2

#### 2. **学生学习模型** (`learning`)
- 检测行为：`hand-raising`（举手）、`read`（阅读）、`write`（书写）
- 类别ID：3-5

#### 3. **学生讨论模型** (`discussion`)
- 检测行为：`discuss`（讨论）
- 类别ID：6

#### 4. **综合场景模型** (`comprehensive`)
- 检测行为：
  - 教师行为：`guide`（引导）、`answer`（回答）、`teacher`（授课）、`blackboard-writing`（板书）、`stand`（站立）
  - 互动行为：`On-stage interaction`（上台互动）
  - 场景元素：`screen`（屏幕）、`blackBoard`（黑板）
- 类别ID：7-14

**检测过程：**
```python
# 对每一帧进行检测
for frame in video_frames:
    detections = detector.detect(frame)
    # 返回格式：
    # [
    #   {"category": "hand-raising", "bbox": [x1,y1,x2,y2], "score": 0.95},
    #   {"category": "read", "bbox": [x1,y1,x2,y2], "score": 0.87},
    #   ...
    # ]
```

**检测结果包含：**
- `category`: 行为类别（如 "hand-raising"）
- `bbox`: 检测框坐标 [x1, y1, x2, y2]
- `score`: 置信度（0-1）

---

### 第四步：时间窗口聚合

**窗口大小：** 10秒

**聚合逻辑：**
```python
# 将检测结果按10秒窗口聚合
window_key = (timestamp // 10) * 10  # 例如：15秒 → 10秒窗口

# 统计每个窗口内的行为数量
window_data[window_key] = {
    "behavior_counts": {
        "hand-raising": 3,    # 该窗口内检测到3次举手
        "read": 15,           # 该窗口内检测到15次阅读
        "discuss": 2,         # 该窗口内检测到2次讨论
        ...
    },
    "detections": [...]      # 原始检测数据（可选存储）
}
```

**为什么使用10秒窗口？**
- ✅ 平滑噪声，避免单帧误检
- ✅ 保持时间分辨率，能捕捉行为变化
- ✅ 数据量适中，便于存储和查询

---

### 第五步：全局统计计算

**统计指标：**

#### 1. **行为汇总统计**
对每个行为类别计算：
```python
behavior_summary[category] = {
    "count": 总检测次数,              # 整个视频中检测到的总次数
    "total_duration": 出现的秒数,      # 该行为出现的独立时间点数量
    "percentage": 占比百分比          # (出现秒数 / 总时长) × 100%
}
```

**示例：**
- `hand-raising`: count=45, total_duration=30秒, percentage=5%
  - 含义：整节课检测到45次举手，出现在30个不同的时间点，占总时长的5%

#### 2. **关键指标计算**

**互动率 (interaction_rate)**
```python
互动率 = 讨论时长 / 总时长
```
- 衡量课堂互动程度

**专注度 (attention_rate)**
```python
专注度 = 1 - (低头时长 / 总时长)
```
- 衡量学生注意力集中程度
- 低头时长越少，专注度越高

**参与度评分 (engagement_score)**
```python
参与度 = min(1.0, (互动率 + 举手次数/50) / 2)
```
- 综合评估学生参与度
- 范围：0-1

---

### 第六步：异常检测

**检测两种异常：**

#### 1. **持续低头异常**
```python
# 检测逻辑
if 低头率 > 30% 且 持续 >= 3个窗口(30秒):
    标记为异常
    严重程度：
        - 持续 >= 6个窗口(60秒) → "high"
        - 持续 3-5个窗口 → "medium"
```

#### 2. **互动不足异常**
```python
# 检测逻辑
if 连续 >= 6个窗口(60秒) 无讨论且无举手:
    标记为异常
    严重程度：medium
```

**异常数据格式：**
```python
{
    "start_time": 120,      # 异常开始时间（秒）
    "end_time": 180,        # 异常结束时间（秒）
    "type": "high_bowhead_rate",
    "severity": "high",
    "description": "低头率持续偏高（>30%），持续60秒",
    "behavior_stats": {...}
}
```

---

### 第七步：数据存储

**存储到数据库：**

1. **时间线数据表** (`analysis_timeline`)
   - 每个10秒窗口一条记录
   - 存储：时间戳、行为计数、检测详情

2. **汇总数据表** (`analysis_summary`)
   - 每个视频一条记录
   - 存储：行为汇总、关键指标

3. **异常数据表** (`analysis_anomalies`)
   - 每个异常时段一条记录
   - 存储：时间范围、异常类型、严重程度

## 📊 前端展示逻辑

### API 接口

1. **`GET /analysis/{video_id}/summary`**
   - 返回整课行为统计汇总
   - 包含：行为分布、教师行为、关键指标

2. **`GET /analysis/{video_id}/timeline`**
   - 返回时间序列数据
   - 参数：`window` (10秒或60秒)
   - 用于绘制时间趋势图

3. **`GET /analysis/{video_id}/anomalies`**
   - 返回异常时段列表
   - 参数：`threshold` (Z-score倍数，默认2.0)

4. **`GET /analysis/{video_id}/causation`**
   - 返回行为相关性分析
   - 目前使用预定义规则（可改进为真实计算）

## 🎯 分析特点

### ✅ 优势

1. **多模型融合**
   - 4个专门模型，检测精度高
   - 覆盖学生、教师、场景全方位行为

2. **时间窗口聚合**
   - 平滑噪声，提高稳定性
   - 保持时间分辨率

3. **异常检测**
   - 自动识别问题时段
   - 帮助教师快速定位

4. **异步处理**
   - 不阻塞用户操作
   - 支持长时间视频分析

### ⚠️ 当前限制

1. **相关性分析简化**
   - 目前使用预定义规则
   - 未使用真实统计方法（如皮尔逊相关系数）

2. **异常检测规则固定**
   - 阈值硬编码（30%低头率、60秒无互动）
   - 未考虑不同课程类型的差异

3. **个体分析缺失**
   - 无法识别具体学生
   - 只能统计整体行为

4. **检测精度依赖模型**
   - 受YOLO-vHeat模型性能限制
   - 可能存在误检或漏检

---

## 🔧 可改进方向

1. **真实相关性计算**
   - 使用 numpy/scipy 计算皮尔逊相关系数
   - 分析教师行为与学生行为的滞后关系

2. **动态阈值**
   - 根据历史数据自适应调整异常阈值
   - 考虑课程类型、年级等因素

3. **更丰富的统计指标**
   - 行为转换频率
   - 峰值时段识别
   - 教学节奏分析

4. **个体识别**（如果数据支持）
   - 结合人脸识别或座位信息
   - 提供学生个体分析

---

## 📝 总结

当前行为分析采用 **"抽帧 → AI检测 → 窗口聚合 → 统计分析 → 异常检测"** 的流程，核心是使用4个YOLO-vHeat模型对视频帧进行多类别行为检测，然后通过时间窗口聚合和统计分析，生成整课行为报告和异常时段。

**核心优势：** 多模型融合、时间窗口平滑、自动异常检测
**主要限制：** 相关性分析简化、异常阈值固定、无个体分析
