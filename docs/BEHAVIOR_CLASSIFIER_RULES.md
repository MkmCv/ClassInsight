# 行为分类器规则实现说明

## 实现内容

我实现了一个**行为分类器** (`behavior_classifier.py`)，将检测器的原始输出映射到您定义的高级行为类别。

## 核心功能

### 1. **单一行为优先原则**
- 每帧只输出一个主导行为（学生行为和教师行为分别输出）
- 如果存在复合动作，按优先级选择主导行为

### 2. **关键区别点的实现**

#### 学生行为的关键区别

| 行为 | 关键区别 | 实现方式 |
|------|---------|---------|
| **台上展示 vs 台上互动** | 是否有教师参与 | 台上展示：排除 `teacher`<br>台上互动：需要 `teacher` |
| **回答问题 vs 应答** | 是否有教师在场 | 回答问题：排除 `teacher`<br>应答：需要 `teacher` + `answer` + `stand` |
| **朗读 vs 读写** | 是否有张嘴动作 | 朗读：优先级高于读写（当前基于 `read` 推断，无法检测张嘴） |
| **学生板书 vs 教师板书** | 谁在板书 | 学生板书：排除 `teacher`<br>教师板书：需要 `teacher` |
| **学生举手** | 需要3人及以上 | `min_count: 3`，统计时按事件计数 |

#### 教师行为的关键区别

| 行为 | 关键区别 | 实现方式 |
|------|---------|---------|
| **讲授 vs 应答** | 是否有学生站立回答问题 | 讲授：排除 `answer` 和学生 `stand`<br>应答：需要 `teacher` + `answer` + `stand` |
| **指导 vs 巡视** | 是否有弯腰、驻足等动作 | 指导：需要 `guide`<br>巡视：只有 `teacher`（当前无法检测弯腰、驻足） |
| **台上互动 vs 台上展示** | 是否有教师参与 | 台上互动：需要 `teacher`<br>台上展示：排除 `teacher` |

### 3. **分类规则结构**

每个行为类别都有以下规则：

```python
"行为名称": {
    "required": ["必需类别1", "必需类别2"],  # 必须同时存在
    "exclude": ["排除类别1"],                  # 不能存在
    "context_required": ["上下文要求"],         # 上下文必须存在
    "context_exclude": ["上下文排除"],         # 上下文不能存在
    "min_count": 3,                            # 最小数量要求（如学生举手）
    "priority": 1                              # 优先级（数字越小优先级越高）
}
```

### 4. **分类流程**

1. **统计检测结果**：统计每帧检测到的类别及其数量
2. **检查必需类别**：确保所有必需类别都存在
3. **检查排除类别**：确保没有排除的类别
4. **检查上下文要求**：验证上下文条件
5. **检查数量要求**：如学生举手需要≥3人
6. **计算匹配分数**：基于优先级和检测数量
7. **选择最高分**：单一行为优先

## 当前实现的局限性

### 无法准确区分的场景

1. **朗读 vs 读写**
   - 需要：嘴部检测（张开嘴）
   - 当前：基于 `read` 推断，优先级高于读写
   - 建议：添加嘴部检测模型

2. **指导 vs 巡视**
   - 需要：姿态估计（弯腰、驻足）
   - 当前：基于 `guide` 检测，无法识别细微动作
   - 建议：添加姿态估计模型

3. **教师站立 vs 学生站立**
   - 问题：`stand` 类别无法区分是教师还是学生
   - 当前：基于上下文推断（如是否有 `teacher`、`answer` 等）
   - 建议：使用 bbox 重叠判断或添加角色识别

## 数据统计逻辑

### 学生举手特殊处理

- **分类时**：只有检测到≥3个 `hand-raising` 才分类为"学生举手"
- **统计时**：
  - 全局统计：间隔超过5秒算新事件（避免连续帧重复计数）
  - 窗口统计：每个10秒窗口最多计数1次举手事件
  - 结果：`count` 表示举手事件次数，不是举手人数

## 使用方式

分类器已自动集成到视频处理流程中：

1. **上传视频** → 系统自动检测
2. **检测结果** → 自动分类为高级行为类别
3. **存储数据** → 同时保存原始和分类后的数据
4. **前端展示** → 优先显示分类后的行为（中文类别名）

## 验证方法

1. **查看分类结果**：上传新视频，查看行为分析页面
2. **检查数据格式**：确认同时有原始和分类后的数据
3. **验证关键区别**：
   - 检查"回答问题"和"应答"是否正确区分
   - 检查"台上展示"和"台上互动"是否正确区分
   - 检查"学生举手"是否按事件计数（3人及以上算1次）

## 优化建议

1. **短期**：根据实际数据调整优先级和规则
2. **中期**：添加嘴部检测和姿态估计
3. **长期**：使用机器学习模型优化分类规则
