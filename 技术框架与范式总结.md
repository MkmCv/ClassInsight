# ClassInsight 课堂行为分析系统 - 技术框架与范式总结

## 📋 项目概述

ClassInsight 是一个基于 AI 的课堂行为智能分析系统，采用前后端分离架构，使用 Python 全栈技术栈构建。系统集成了 YOLO-vHeat 多模型检测、LangGraph AI Agent 等先进技术，实现了从视频上传、行为检测、数据分析到教学建议的完整闭环。

- **前后端分离**: 前端通过 RESTful API 与后端通信

- **分层架构**: 清晰的 API → Service → Model 三层架构
- **依赖注入**: FastAPI 的 Depends 机制实现依赖注入
- **模块化设计**: 按功能模块划分（认证、视频、分析、优化等）

#### 3.2.2 依赖注入模式 (Dependency Injection)

FastAPI 通过 `Depends` 实现依赖注入：

```python
    async with AsyncSessionLocal() as session:
        yield session

# 用户认证注入
async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: AsyncSession = Depends(get_db)
) -> User:
    # 验证 Token 并返回用户对象
    ...

# 在端点中使用
@router.get("/me")
async def get_current_user_info(
    current_user: User = Depends(get_current_user)
):
    return UserResponse.model_validate(current_user)
```

## 🤖 四、AI/ML 技术栈

### 4.1 计算机视觉

#### 4.1.1 YOLO-vHeat 多模型检测

系统集成了 4 个专门的 YOLO-vHeat 模型：

| 模型 | 检测类别 | 全局ID范围 |
|------|---------|-----------|
| **comprehensive_scene** | 教师行为、场景元素 | 7-14 |
| **student_learning** | 举手、阅读、书写 | 3-5 |
| **student_discussion** | 讨论 | 6 |
| **student_posture** | 低头、转头 | 1-2 |

**检测流程**:
1. 视频抽帧（默认 1 FPS）
2. 并行调用 4 个模型进行检测
3. 结果融合与去重（NMS）
4. 时间窗口聚合（10秒窗口）
5. 存储到数据库

## 🔄 六、业务流程范式

### 6.1 视频处理流程

```
1. 用户上传视频
   ↓
2. 后端接收文件，存储到 storage/videos/
   ↓
3. 创建 Video 记录（status=pending）
   ↓
4. 触发异步处理任务 process_video_task()
   ↓
5. 更新状态为 processing
   ↓
6. 读取视频元数据（FPS、时长等）
   ↓
7. 抽帧（1 FPS）→ 调用检测器 → 聚合结果
   ↓
8. 存储 AnalysisTimeline、AnalysisSummary、AnalysisAnomaly
   ↓
9. 更新状态为 completed
```

### 6.2 异常检测算法

- **滑动窗口**: 10秒时间窗口
- **阈值检测**: 低头率 > 30% 且持续 3 个窗口以上 → 异常
- **互动检测**: 连续 60 秒无讨论和举手 → 低互动异常

### 6.3 数据分析范式

- **时间序列分析**: 按时间窗口聚合行为数据
- **统计聚合**: 计算整课行为汇总（次数、时长、占比）
- **指标计算**: 
  - 互动率 = 讨论时长 / 总时长
  - 专注度 = 1 - 低头率
  - 参与度 = (互动率 + 举手频率) / 2

## 📝 八、技术选型理由

### 8.1 为什么选择 FastAPI？

- ✅ **高性能**: 基于 Starlette 和 Pydantic，性能接近 Node.js
- ✅ **异步支持**: 原生支持 `async/await`，适合 I/O 密集型任务
- ✅ **自动文档**: 自动生成 OpenAPI/Swagger 文档
- ✅ **类型安全**: 基于 Pydantic 的数据验证

### 8.2 为什么选择 Streamlit？

- ✅ **开发效率**: Python 单语言栈，无需前后端分离开发
- ✅ **数据科学友好**: 原生支持 Pandas、Plotly 等数据工具
- ✅ **快速原型**: 适合快速构建数据可视化应用
- ✅ **易于维护**: 后端算法工程师也能维护前端

### 8.3 为什么选择 SQLAlchemy 异步？

- ✅ **性能**: 异步 I/O 避免阻塞，提高并发能力
- ✅ **兼容性**: 支持 SQLite（开发）和 PostgreSQL（生产）
- ✅ **ORM 优势**: 类型安全、自动迁移、关系管理
